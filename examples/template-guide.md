# Chezmoi Template Authoring Guide

## Why Use Templates

Templates allow you to:
- Have the same config file behave differently on different machines
- Hide sensitive information (using password managers)
- Adjust configs based on OS, hostname, and other conditions

## Basic Syntax

### Variable Interpolation

```
Hello, {{ .name }}!
```

### Conditionals

```
{{ if eq .chezmoi.os "darwin" }}
# macOS config
{{ else if eq .chezmoi.os "linux" }}
# Linux config
{{ else }}
# Other systems
{{ end }}
```

### Loops

```
{{ range .packages }}
install {{ . }}
{{ end }}
```

## Common Variables

### System Info

```
{{ .chezmoi.os }}           # darwin, linux, windows
{{ .chezmoi.arch }}         # amd64, arm64, 386
{{ .chezmoi.hostname }}     # Hostname
{{ .chezmoi.username }}     # Username
{{ .chezmoi.homeDir }}      # Home directory
```

### Custom Data

Define in `chezmoi.toml`:

```toml
[data]
    name = "John Doe"
    email = "john@example.com"
    editor = "nvim"
```

Use in templates:

```
name = {{ .name }}
email = {{ .email }}
```

## Practical Examples

### Example 1: Git Config

**Goal**: Use different emails based on machine type

`dot_gitconfig.tmpl`:

```
[user]
    name = {{ .name }}
{{ if eq .chezmoi.hostname "work-laptop" }}
    email = {{ .work.email }}
    signingkey = {{ .work.gpg_key }}
{{ else }}
    email = {{ .email }}
    signingkey = {{ .personal.gpg_key }}
{{ end }}

[core]
    editor = {{ .editor }}
    autocrlf = {{ if eq .chezmoi.os "windows" }}true{{ else }}input{{ end }}

[init]
    defaultBranch = main

[includeIf "gitdir:~/work/"]
    path = ~/.gitconfig-work
```

Matching `chezmoi.toml`:

```toml
[data]
    name = "Your Name"
    email = "personal@example.com"
    editor = "vim"

[data.personal]
    gpg_key = "ABC123"

[data.work]
    email = "work@company.com"
    gpg_key = "XYZ789"
```

### Example 2: Shell Config

**Goal**: Use different paths on different systems

`dot_zshrc.tmpl`:

```
# Generated by chezmoi
# OS: {{ .chezmoi.os }}
# Host: {{ .chezmoi.hostname }}

# Base paths
export PATH="$HOME/bin:$PATH"

{{ if eq .chezmoi.os "darwin" }}
# macOS config
export PATH="/opt/homebrew/bin:$PATH"
export PATH="/opt/homebrew/sbin:$PATH"

{{ else if eq .chezmoi.os "linux" }}
# Linux config
export PATH="$HOME/.local/bin:$PATH"

{{ end }}

# Editor
export EDITOR={{ .editor | default "vim" }}

{{ if eq .chezmoi.hostname "work-laptop" }}
# Work machine specific config
source ~/.zshrc-work
{{ end }}

# Aliases
alias ll='ls -la'
alias gs='git status'
```

### Example 3: SSH Config

**Goal**: Use different keys based on environment

`private_dot_ssh/config.tmpl`:

```
# Personal GitHub
Host github.com
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_personal
    AddKeysToAgent yes

{{ if eq .chezmoi.hostname "work-laptop" }}
# Work GitHub
Host work-github
    HostName github.company.com
    User git
    IdentityFile ~/.ssh/id_work

# Work servers
Host *.company.com
    User {{ .work.username }}
    IdentityFile ~/.ssh/id_work
    ForwardAgent yes

{{ end }}

# General config
Host *
    ServerAliveInterval 60
    ServerAliveCountMax 3
```

### Example 4: Dynamic Content

**Goal**: Insert generation time in config

```
# {{ .chezmoi.sourceFile }}
# Generated at {{ now | date "2006-01-02 15:04:05" }}
# Do not edit directly
```

### Example 5: Password Manager Integration

**Goal**: Get sensitive info from password manager

```
# GitHub Token
{{ if eq .chezmoi.os "darwin" }}
# macOS Keychain
export GITHUB_TOKEN={{ output "security" "find-generic-password" "-s" "github-token" "-w" }}

{{ else }}
# 1Password
export GITHUB_TOKEN={{ onepasswordRead "op://Personal/GitHub/token" }}

{{ end }}
```

### Example 6: Conditional Include

**Goal**: Load config based on file existence

```
{{ if stat (joinPath .chezmoi.homeDir ".work") }}
# Work environment detection file exists
export WORK_ENV=1
source ~/.zshrc-work
{{ end }}
```

### Example 7: String Processing

**Goal**: Process variable values

```
# Uppercase
export USER_UPPER={{ .chezmoi.username | upper }}

# Lowercase
export EMAIL_LOWER={{ .email | lower }}

# Title case
export NAME_TITLE={{ .name | title }}

# Trim whitespace
export TRIMMED={{ .input | trim }}

# Replace
export DOMAIN={{ .email | replace "@" "_at_" }}

# Default value
export EDITOR={{ .editor | default "vim" }}
```

## Advanced Techniques

### Using Comments

```
{{- /* This is a template comment, won't output to target file */ -}}
```

### Whitespace Trimming

Use `-` to trim surrounding whitespace:

```
{{- if eq .chezmoi.os "darwin" }}
# No empty line before this
{{- end }}
```

### Nested Conditions

```
{{ if eq .chezmoi.os "linux" }}
    {{ if eq .chezmoi.arch "arm64" }}
    # ARM64 Linux
    {{ else }}
    # x86 Linux
    {{ end }}
{{ end }}
```

### Using Variables

```
{{ $os := .chezmoi.os }}
{{ $arch := .chezmoi.arch }}

OS: {{ $os }}
Arch: {{ $arch }}
```

## Debugging Templates

### View Render Result

```bash
# View final rendered config
chezmoi cat ~/.gitconfig

# View template data
chezmoi data

# Verbose output
chezmoi apply -v
```

### Common Errors

| Error | Cause | Solution |
|-----|------|------|
| `unexpected "}}"` | Bracket mismatch | Check `{{` and `}}` pairing |
| `function not defined` | Using non-existent function | Check function name spelling |
| `bad character` | Special char not escaped | Use quotes around strings |

## Best Practices

1. **Static first, template later**: Add static files first, convert to template after testing
2. **Small iterations**: Modify one template at a time for easier debugging
3. **Use cat to verify**: `chezmoi cat` to view render result
4. **Comment**: Add generation info in templates for debugging
5. **Default values**: Use `default` function for fallback values

## Quick Reference

```bash
# Add as template
chezmoi add --template ~/.gitconfig

# Or manually rename
cd $(chezmoi source-path)
mv dot_gitconfig dot_gitconfig.tmpl

# View render result
chezmoi cat ~/.gitconfig

# Apply template
chezmoi apply ~/.gitconfig
```
